### NeetCode - Algorithms and Data Structures for Beginners

やりました。Arrays、Linked Listsあたりから始まって、BSTやグラフの探索、DPなどの一通りって感じですね。もっと難しいトピックはAdvanced Algorithmsのほうに入ってるようなのですが、NeetCode（というかLeetCode？）のEasyやMediumを解くベースとしては十分っぽいです。

* 動画を見る（優しい英語なので雰囲気で聞き取る）
* 問題を解く
* 半分くらいはわからん、写経する

みたいなサイクルだったと思います。動画の内容でつまづくことはなかったし、問題も癖がないので学んだことがすぐ活かせる感じでよかったです。回答読むのめんどくさいなーってときも解説動画があって、ぼーっと眺めてれば良いのが助かります。

とはいえ、ちゃんと問題が解けるようになったわけではないので、今後訓練していきたいですね。復習して3日後くらいに解き直すとよいとClaudeが言っていました。個人的にやりたいのは、その場で解説を読むだけじゃなくて、「読む」→「その場ですぐ白紙に実装できるか再チャレンジ」ですね。読むだけだと頭に残らない。。。

想定が25時間って書いてありますけど、知ってる概念も多かったので10~15時間くらいでやれたんじゃないかなと思います。測ってないけど。

### Write a Compiler That Understands Types を見つつ実装する

この講座めっちゃよかったですね。これのためだけにFrontend Mastersに課金して、見終わったら退会しました。退会するの忘れそうなので。

概要は[コンパイラ周りの勉強がしたくて](/blog/2026-05-03-compilers)に書いたので割愛しますが、型DBを使ってUnification（単一化）することで、型推論や型チェックができる部分はだいぶ面白かったです。型推論という抽象的なものの実装イメージが湧いた感じでした。そういえば、WASMへのCode Generate部分はほぼ記憶ないな。

該当リポジトリには字句解析からCode Generationまで一通り揃ってるので、忘れた頃にまた眺めつつ、自分で実装できるか試したいですね。とても貴重な学びでした。

### Simple but Powerful Pratt Parsing を実装する

かなり短いコードなのに、パーサーになってすげー！という感動がありました。めっちゃ前に眺めて何も覚えていなかった[Write a JavaScript Parser in Rust](https://oxc-project.github.io/javascript-parser-in-rust/ja/)にも参考文献として書かれていて、繋がったなーと一人で驚いていました。今回それを理解できたのは嬉しいですね。

メモった[スクラップ](https://zenn.dev/sushin_ya/scraps/d789d17b98afcf)

### acorn の実装を眺める

眺めました。[スクラップ](https://zenn.dev/sushin_ya/scraps/634eef6e351448)。

TokenizerとParser合わせても数千行なので、全然読めました。しかし、ただ読むのはやっぱり退屈で雑にClaudeに解説してもらって、へえーつって終わりましたね。

[JS syntactic quirks](https://github.com/mozilla-spidermonkey/jsparagus/blob/master/js-quirks.md)などに書かれているような曖昧な文法だったりをどうやって対処してるんだろうと思ったのですが、再帰下降だと取り回しがしやすいのか、結構素直に（？）書かれてるんだなと驚きました。もっと謎のテクニックを使ってるかと思ってましたが、案外読めましたね。例えば`/`が除算か正規表現か、`(`がアロー関数か引数リストか、などなど。

これもいざ自分で実装しろって言われたら全然できなさそうなので、小さいサブセットから始めてやってみたいなあと思ったりしました。やるならTypeScriptのサブセットにしたい気持ちもありますが大変だってClaudeが言ってた。

### Martin Grabmüller - Algorithm W Step by Step を読みつつ実装する

Milnerの提案したAlgorithm Wを実装することで、型推論について理解しようという論文です。論文というかほぼコード片が書いてあるだけで、なぜそうなってるかはClaudeに解説してもらわないと何もわかりませんでした。[スクラップ](https://zenn.dev/sushin_ya/scraps/4b0caeebad7cfb)。

Write a Compiler That Understands Typesの型推論とほぼ同じようなことをしてるっぽいが、Haskellの抽象度が高いコードで再現されていてすげーって感じでした。あとモナドトランスフォーマーってなんだろうと思ってたので、具体例としてわかりやすかったです。

コードは抽象度が高すぎて雰囲気しかわからなかったのですが、直近で勉強していたことがつながった感じがして面白かったです。[Typechecker Zoo](https://sdiehl.github.io/typechecker-zoo/introduction.html)にはRust実装もあるらしいので、そのうち読みたい気持ちです。
